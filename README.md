## üìÅ Context & positioning

This dashboard is **not a standalone project**. It is designed to live inside a larger AI working directory ‚Äî a workspace where you collaborate with an AI CLI (Claude Code, Gemini CLI, or similar) to manage a WordPress site.

The workspace structure looks like this:

```

‚îÇ
‚îú‚îÄ‚îÄ PLAYBOOK.md                ‚Üê Session memory + validated workflows
‚îú‚îÄ‚îÄ package.json               ‚Üê npm run dashboard
‚îÇ
‚îú‚îÄ‚îÄ .claude/                   ‚Üê Claude Code config (MCP, settings‚Ä¶)
‚îÇ
‚îú‚îÄ‚îÄ dashboard/  ‚Üê ‚Üê ‚Üê ‚Üê ‚Üê    THIS REPO ‚Äî cloned here
‚îÇ   ‚îú‚îÄ‚îÄ server.js
‚îÇ   ‚îú‚îÄ‚îÄ config.js              (gitignored ‚Äî your local values)
‚îÇ   ‚îî‚îÄ‚îÄ ‚Ä¶
‚îÇ
‚îú‚îÄ‚îÄ wiki/                      ‚Üê Markdown export of your WP site
```

## üñºÔ∏è Unified Modal Viewer & Editor

The dashboard features a centralized, intelligent preview system. Instead of opening new tabs, all files are opened in a premium dark-themed modal:

*   **üìù Markdown**: Fully rendered HTML preview using `marked.js`.
*   **üíª Code Highlighting**: Syntax highlighting for `JS`, `HTML`, `CSS`, `JSON`, `PHP`, `SQL`, and more via `Prism.js`.
*   **üìë Documents**: Integrated `Iframe` viewer for `PDF` and `HTML` files (perfect for CVs and reports).
*   **üñºÔ∏è Media Gallery**: Centered preview for `PNG`, `JPG`, `SVG`, `WebP`, and `GIF`.
*   **‚úèÔ∏è In-App Editing**: Edit any text-based file directly in the modal and save changes to your local disk with one click.
*   **‚ÜóÔ∏è Quick Access**: Shortcut button to open the raw file in a new tab if needed.

## üõÝÔ∏è Installation

### Prerequisites

This project expects to sit inside a workspace directory that contains sibling folders like `wiki/`, `cv/`, `skills/`, `scripts/`. See [`examples/workspace-structure.md`](examples/workspace-structure.md) for the full recommended layout.

If you just want to try it without a full workspace, a minimal setup works too ‚Äî most widgets will simply show as empty.

### 1. Clone inside your workspace

```bash
cd your-claude-workspace/
git clone https://github.com/molokoloco/ia-dashboard-pro.git dashboard
cd dashboard
npm install
```

### 2. Configure

```bash
cp config.example.js config.js
```

Edit `config.js` with your values:

```js
module.exports = {
  port: 3001,

  // WordPress REST API ‚Äî create an Application Password in WP Admin
  wpBase: 'https://yoursite.com/wp-json/wp/v2',
  wpUser: 'your_wp_username',
  wpPassword: 'xxxx xxxx xxxx xxxx xxxx xxxx',

  // Files to show in the Docs widget (paths relative to parent directory)
  docsFiles: [
    { rel: 'NOTES.md', tags: ['notes'] },
  ],

  // Explorer widget ‚Äî which directories to expose
  explorerDirs: {
    'my-folder': { type: 'code', icon: 'üìÅ', label: 'My Folder', viewer: 'dir' },
  },

  // Widgets to display
  widgets: [ /* see config.example.js for full list */ ],
};
```

> ‚öÝÔ∏è **`config.js` is gitignored.** It contains your credentials and personal paths ‚Äî never commit it.

### 3. Run

```bash
node server.js
# or
npm start
```

Open **http://localhost:3001**

### Parent directory dependencies

| Widget | Reads from | Behaviour if missing |
|--------|-----------|----------------------|
| Wiki | `../wiki/articles/` + `../wiki/pages/` | Empty list |
| CV | `../cv/` (served as `/static/cv`) | Empty list |
| Skills | `../skills/` | Empty list |
| Scripts | `../scripts/` | Empty list |
| Explorer | Dirs defined in `config.js` ‚Üí `explorerDirs` | Widget hidden |
| Docs | Files defined in `config.js` ‚Üí `docsFiles` | Empty list |
| WordPress | `config.js` ‚Üí `wpBase` + credentials | Error in widget |
| Todos / Sessions / Candidatures | `dashboard.db` (SQLite, auto-created) | Always works |

## üîå API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/todos` | List todos |

| GET | `/api/wordpress/pages` | WordPress pages (live) |
| GET | `/api/wiki` | Local Markdown wiki articles |
| GET | `/api/docs` | Markdown docs + PDF index |
| GET | `/api/skills` | Claude Skills catalog |
| GET | `/api/explorer` | Directory listing |

## üîí Security notes

- **Local use only.** This dashboard is designed to run on `localhost`. Do not expose it on a public network.
- **`api/explorer.js`** allows browsing, reading, and **writing** files from the parent directory tree. This is a local development tool; ensure your project root is safe.
- `config.js` is gitignored. Your credentials stay on your machine.

## 4. WAF CONSTRAINTS — GOLDEN RULES

| Constraint | Symptom | Solution |
|-----------|----------|----------|
| Missing User-Agent | 403 on all requests | Always send `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36` |
| DELETE method blocked | 403 on DELETE | **Manual** deletion in WP Admin (snippets, tags, media) |
| Raw binary media upload | 403 | Use **multipart/form-data** with boundary |
| WP Super Cache | Schemas/metas not visible | Append `?nocache=timestamp` to check · Manual purge in WP Admin |

```js
// Node.js request template (base for all scripts)
const https = require('https');
const AUTH = Buffer.from('your_wp_username:xxxx xxxx xxxx xxxx xxxx xxxx').toString('base64');
const UA = 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36';

function req(method, path, body) {
  return new Promise((resolve, reject) => {
    const bodyStr = body ? JSON.stringify(body) : null;
    const options = {
      hostname: 'yoursite.com', path, method,
      headers: {
        'Authorization': 'Basic ' + AUTH, 'User-Agent': UA,
        'Content-Type': 'application/json',
        ...(bodyStr ? { 'Content-Length': Buffer.byteLength(bodyStr) } : {})
      }
    };
    const r = https.request(options, res => {
      let d = ''; res.on('data', c => d += c);
      res.on('end', () => { try { resolve({ status: res.statusCode, data: JSON.parse(d) }); } catch(e) { resolve({ status: res.statusCode, data: d }); } });
    });
    r.on('error', reject);
    if (bodyStr) r.write(bodyStr);
    r.end();
  });
}
```

## 6. CODE SNIPPETS — Universal pattern

The **Code Snippets** plugin (REST API: `/wp-json/code-snippets/v1/snippets`) is the Swiss army knife for everything the WP API doesn't expose natively: Yoast metas, Elementor post_meta, cache purge, etc.

### Full pattern
```js
// 1. Create snippet (inactive)
const create = await req('POST', '/wp-json/code-snippets/v1/snippets', {
  name: 'TMP_MY_ACTION',
  code: `<?php add_action('init', function() { /* PHP here */ });`,
  scope: 'front-end',
  active: false,
});
const id = create.data.id;

// 2. Activate
await req('POST', `/wp-json/code-snippets/v1/snippets/${id}/activate`, null);

// 3. Trigger execution (WP init hook)
await req('GET', '/wp-json/wp/v2/posts?per_page=1', null);

// 4. Deactivate
await req('POST', `/wp-json/code-snippets/v1/snippets/${id}/deactivate`, null);

// 5. ⚠️ Delete MANUALLY in WP Admin (DELETE blocked by WAF)
```

> **⚠️ PHP in snippets:** Always escape apostrophes in PHP single-quoted strings with `\'`. Ex: `'it\'s'`, `'don\'t'`. Unescaped apostrophes silently break PHP without a detectable API error.

### Validated use cases

| Action | Method |
|--------|---------|
| Edit `_yoast_wpseo_metadesc` | Code Snippets + `update_post_meta` |
| Edit `_yoast_wpseo_focuskw` | same |
| Inject Elementor metas | same (`_elementor_edit_mode`, `_elementor_template_type`, `_elementor_page_template`) |
| Purge WP Super Cache | Code Snippets + `wp_cache_clear_cache()` |
| Edit any private post_meta (prefix `_`) | Code Snippets (not natively exposed) |

## 8. ARTICLE PUBLICATION

```
1. Local Markdown → HTML via marked.parse()
   (remove the H1 from markdown before conversion)

2. Resolve categories and tags
   GET /wp-json/wp/v2/categories?search=NAME&per_page=5
   GET /wp-json/wp/v2/tags?search=NAME&per_page=5
   POST /wp-json/wp/v2/tags { name } // create if missing

3. Upload image (MULTIPART mandatory)
   const boundary = '----FormBoundary' + Math.random().toString(36).substring(2);
   POST /wp-json/wp/v2/media (Content-Type: multipart/form-data; boundary=...)
   → returns { id, source_url }

4. Create draft
   POST /wp-json/wp/v2/posts {
     title, content (HTML), excerpt, slug,
     status: 'draft',
     categories: [catId], tags: [tagIds],
     featured_media: mediaId
   }
   → returns { id, link }

5. Add image alt text
   POST /wp-json/wp/v2/media/:id { alt_text, title, caption }

6. Inject Elementor metas (CRITICAL — otherwise no sidebar)
   update_post_meta($id, '_elementor_edit_mode',       'builder');
   update_post_meta($id, '_elementor_template_type',   'post');
   update_post_meta($id, '_elementor_page_template',   'theme'); // ← key!

7. Edit in WP Admin: /wp-admin/post.php?post=ID&action=edit
```

**Why `_elementor_page_template = "theme"` is critical:**
Without this meta, Elementor doesn't apply the Theme Builder "Single Article" → no sidebar, broken layout.

**Article heading hierarchy:**
- H2 → article title (first element of content)
- H3 → main sections
- H4 → subsections

# PART V — FIELD REFERENCE

## 15. WATCHPOINTS & LESSONS LEARNED

### PHP in snippets
- Always escape `'` → `\'` in PHP single-quoted strings
- The Code Snippets API validator (`code_error`) doesn't always catch runtime errors
- Test with `?nocache=timestamp` to bypass WP Super Cache

### Yoast SEO
- Private metas (prefix `_`) not natively exposed → Code Snippets mandatory
- Orange score = keyword missing from one of the three locations (title / meta desc / content)
- Check: `GET /wp-json/wp/v2/posts/ID?_fields=yoast_head_json`

### WP Super Cache
- Cache invalidated by visit or manual purge
- DELETE on cache endpoint → 403 (WAF)
- Workaround: Code Snippets + `wp_cache_clear_cache()`
- To verify schemas/metas without cache: append `?nocache=` + timestamp

### Elementor
- `_elementor_page_template = "theme"` is critical for Theme Builder
- Without this meta: no sidebar, broken template
- **NEVER inject `_elementor_data` via REST API or Code Snippets PHP** → corruption
- Elementor import: only via the manual UI

### Node.js on Windows
- No shell heredoc → use the `Write` tool to create .js files
- Avoid `node -e` for complex code with nested quotes
- `/dev/stdin` doesn't exist → use temp files

## 18. GIT — PURGING A SECRET

If a token/password is accidentally committed:

```bash
# 1. Install git-filter-repo
pip install git-filter-repo

# 2. Create replacement file
echo "THE_SECRET_TOKEN==>***REMOVED***" > /tmp/replacements.txt

# 3. Purge entire history
python path/to/git_filter_repo.py \
  --replace-text /tmp/replacements.txt --force

# 4. Re-add remote (filter-repo removes it)
git remote add origin https://github.com/your-handle/your-repo.git

# 5. Force-push (overwrites remote history)
git push --force origin main
```

**After purge:** revoke and regenerate the token on the relevant platform.

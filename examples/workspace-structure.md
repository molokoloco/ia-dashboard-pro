# Workspace Structure — Example

This document shows the recommended folder layout for using `ia-dashboard-pro`
as part of a larger AI + WordPress working environment.

The dashboard (`dashboard/`) is **not a standalone project** — it is one component
inside a broader workspace where you collaborate with Claude Code (or any AI CLI)
to manage a WordPress site.

---

## Recommended tree

```
Claude/                              ← Your main AI working directory (any name)
│
├── PLAYBOOK.md                      ← Your own session memory + workflows (gitignored)
├── package.json                     ← { "scripts": { "dashboard": "node dashboard/server.js" } }
│
├── .claude/                         ← Claude Code config
│   ├── settings.json                ←   MCP servers, permissions
│   └── commands/                    ←   Custom slash commands
│
├── dashboard/                       ← ★ THIS REPO (cloned here)
│   ├── server.js
│   ├── config.js                    ←   Your local config (gitignored)
│   ├── config.example.js            ←   Template (committed)
│   └── …
│
├── wiki/                            ← Markdown export of your WP site
│   ├── articles/                    ←   YYYY-MM-DD_slug.md
│   ├── pages/                       ←   slug.md
│   └── prompts/                     ←   Your AI prompt library
│
├── articles/                        ← Draft articles (.md, in progress)
│
├── exports-elementor/               ← Elementor JSONs generated by AI
│   ├── current/                     ←   Active versions, ready to import
│   ├── lib/                         ←   Reusable templates
│   └── archive/
│
├── medias/                          ← Visual assets (images, videos…)
│
├── scripts/                         ← Node.js WP automation scripts
│   ├── export_wiki.js               ←   WP → local Markdown export
│   ├── create_PAGE_NAME.js          ←   Create WP page + inject metas
│   └── generate_PAGE_NAME.js        ←   Generate Elementor JSON
│
├── skills/                          ← Additional Claude Code skills (.md)
│
├── cv/                              ← CVs + cover letters (optional)
│
└── Projects/                        ← Active projects (one subfolder each)
    └── my-new-page/
        └── my-new-page.html         ← HTML design source
```

---

## What the dashboard reads from the parent directory

| Widget | Reads from | Config key |
|--------|-----------|------------|
| Explorer | Any dirs you configure | `explorerDirs` in `config.js` |
| Wiki | `../wiki/` | hardcoded in `api/wiki.js` |
| CV | `../cv/` | served as `/static/cv` |
| Skills | `../skills/` | `api/skills.js` |
| Scripts | `../scripts/` | `api/scripts.js` |
| Docs | paths you define | `docsFiles` in `config.js` |

All missing directories are handled gracefully (empty widget or 404) — the dashboard
won't crash if a folder doesn't exist yet.

---

## Minimal setup (no WordPress)

If you just want the dashboard without a WordPress site, you only need:

```
Claude/
├── package.json          (with "dashboard" script)
├── dashboard/            (this repo)
└── wiki/                 (optional — create empty articles/ + pages/)
```

The WordPress widget will simply show an error if `wpBase` is not configured.
Todos, sessions, candidatures, and the explorer work fully offline with SQLite.

## How to clone into the right place

```bash
# From your Claude/ working directory:
git clone https://github.com/molokoloco/ia-dashboard-pro.git dashboard
cd dashboard
npm install
cp config.example.js config.js
# Edit config.js with your values
node server.js
```

See [PLAYBOOK.example.md](PLAYBOOK.example.md) for the full workflow reference
(credentials, WAF rules, Elementor patterns, SEO automation…).
